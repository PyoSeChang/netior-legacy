# Product

Product 문서는 Netior가 어떤 제품인지, 어떤 문제를 풀고자 하는지, 어떤 방향으로 성장해야 하는지 정리하는 공간이다.

`philosophy`는 별도 최상위 폴더로 두지 않는다. Netior의 철학은 제품 판단의 일부이므로 `00-product` 안에 둔다.

## Roadmap

Roadmap은 plan과 다르다.

- Roadmap은 제품이 어디로 가야 하는지에 대한 미래지향적 문서다.
- Plan은 지금 무엇을 어떤 순서로 구현할지에 대한 실행 문서다.
- Roadmap은 방향과 단계 감각을 준다.
- Plan은 범위, 완료 조건, 검증 기준을 준다.

## Writing Rule

Product 문서는 구현 세부보다 제품의 정체성, 사용자 가치, 책임 경계, 장기 방향을 먼저 다룬다.

사용자의 교정이나 Codex의 추론을 제품 원칙으로 섞지 않는다. 명시적으로 결정되지 않은 내용은 `미결정` 또는 `검토 필요`로 남긴다.
