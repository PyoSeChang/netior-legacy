# Netior Manifesto Draft

Netior begins from a simple dissatisfaction:
the dominant interfaces for knowledge on computers were designed for storage before they were designed for thought.

Folders and trees are powerful tools for deterministic organization.
They are good at putting something somewhere.
They are much weaker at expressing how things relate, why they matter, and which viewpoint should be active for a given task.

Human knowledge work is not primarily hierarchical.
It is relational.
Meaning emerges through association, context, and changing perspective.
What matters is not just where something is stored, but how it connects, what it depends on, and under which situation it becomes relevant.

Netior takes this seriously.

That is why it chooses canvas, even with all the cost and difficulty that choice brings.
Canvas is not a finished answer.
It is simply a better starting point than the folder tree when the goal is to work with concepts, relations, and shifting context rather than fixed addresses.

Netior is not trying to become just another note app, whiteboard, or graph view.
A graph full of dots and lines is not knowledge.
Visible linkage is not enough.
Reference is only one type of relation, and relation count is not the same thing as understanding.
What matters is not the total amount of connection, but which set of relations is activated in a specific situation.

The real problem is context.
In any serious network of knowledge, the total relation space is too large.
No human and no agent can meaningfully act on the whole thing at once.
The important question is always:

Which relations should be active now?

Netior's answer is to move toward personal knowledge ontology.
Not an overly rigid schema that freezes thought into a machine format, but a living structure that can be interpreted, revised, and expanded over time.
Concepts, relation types, schemas, canvases, and contexts should evolve with the project itself.

This matters because AI is not weak only because it lacks files.
It is weak because it often lacks the right causal model, the right task boundary, and the right viewpoint.
A senior engineer or domain expert can often outperform an agent not by reading more raw data, but by selecting the correct frame of reasoning.

Netior exists to make that frame transferable.

Its ambition is to become the layer through which humans and agents share structure:

- human thinking can be externalized into a navigable ontology
- agents can receive better context instead of undifferentiated input
- the same knowledge can be rearranged under different viewpoints
- stable high-level structure can coexist with changing low-level implementation

If that succeeds, software engineering methods can begin to extend into broader knowledge production.
Validation, modularization, dependency reasoning, structured decomposition, and iterative refinement should not be limited to code.
They should also become available for fiction, research, planning, learning, and other domains where structure matters but traditional software tools do not fit.

Netior therefore aims at something larger than a canvas application.
It is an attempt to build an integrated knowledge interface for humans and AI:
one that treats relation as primary, context as essential, and ontology as something that can be personally built, continuously revised, and computationally useful.
